Encryption design — Tier 3 + Tier 4 walkthrough

Workspace master key for confidentiality • per-author signing keys for identity. No public-key infrastructure beyond the per-author signing keys (which never need exchange).

Workspace-level (one per workspace) Per-file (random per save) Per-author (identity) Cryptographic operation

1. The cast

Three categories of secret/public material. None of them require a key-exchange protocol with another party.

Workspace
Workspace passphrase
  • Typed by the user when unlocking the workspace
  • Argon2id derives the workspace master key (MK)
  • Never persisted
  • One per workspace, shared by everyone with read access
Author — Omer
Ed25519 signing keypair
  • Private signing key — stays on Omer's device
  • Public verify key — embedded in every file Omer signs
  • No exchange protocol; verify keys travel inside files
Author — Claude
Ed25519 signing keypair
  • Private signing key — stays in agent's runtime
  • Public verify key — embedded in every file Claude signs
  • Independent from Omer's keypair

2. Anatomy of an encrypted .gmd file

The file is a single binary blob. Plaintext metadata at the top, ciphertext body, signature footer.

Magic
"GMD1" + version byte
format identifier
Algo IDs
aead=XChaCha20-Poly1305, kdf=Argon2id, sig=Ed25519, comp=zstd
version-friendly
Wrapped CEK
AEADMK(CEK) wrap
CEK encrypted by workspace master key. One wrap, not N.
Salt
random 16 B (Argon2id salt for MK)
in-file so opener can re-derive MK
Nonce
random 24 B (XChaCha20 nonce)
Body
AEADCEK( zstd(plaintext markdown) )
the actual encrypted writing
Author
label="Omer", verify_key=Ed25519 public
readable to anyone (it's identity, not secret)
Signature
Ed25519Omer-priv( hash of everything above )
proves "Omer wrote this version"

3. Save flow — Omer writes a note

1
Workspace already unlocked workspace
MK = Argon2id(passphrase, workspace_salt). MK lives in memory only. If workspace is locked, prompt for passphrase first.
↓
2
Generate fresh CEK per-save
CEK = 32 random bytes from CSPRNG. New CEK every save.
↓
3
Compress + encrypt body
compressed = zstd(plaintext). body_ct = XChaCha20-Poly1305(key=CEK, nonce, compressed)
↓
4
Wrap CEK with workspace MK
wrapped_CEK = XChaCha20-Poly1305(key=MK, fresh_nonce, CEK). MK never touches the file body — only wraps the small CEK.
↓
5
Sign with Omer's Ed25519 key identity
sig = Ed25519_sign(Omer_priv, hash(magic ‖ algos ‖ wrapped_CEK ‖ salt ‖ nonce ‖ body_ct ‖ author_block))
↓
6
Write file atomically
Concatenate header + wrapped_CEK + salt + nonce + body_ct + author_block + sig. Write to temp + fsync + rename.

4. Open flow — Claude opens that file

1
Read header + retrieve passphrase workspace
Claude has the workspace passphrase too (this tier shares one workspace key). Re-derive MK = Argon2id(passphrase, salt_from_file).
↓
2
Unwrap CEK
CEK = AEAD_decrypt(MK, wrapped_CEK). If decryption fails → wrong workspace passphrase.
↓
3
Decrypt + decompress body
compressed = AEAD_decrypt(CEK, nonce, body_ct). plaintext = zstd_decompress(compressed). AEAD also verifies body wasn't tampered.
↓
4
Verify author signature identity
Read author_block: { label="Omer", verify_key }. Verify sig with verify_key. UI displays: "Last edited by Omer ✓".

5. Identity in practice

What the user sees in the editor

A small badge near the file title:

📝 meeting-notes.gmd
Last saved by Omer · signature verified ✓

Tampering with the body invalidates the signature; UI shows "⚠ signature invalid" instead of the green check.

What about per-paragraph attribution?

Out of scope for v1. The signature is per-file-version, not per-paragraph. If you want "this paragraph was written by Claude, that one by Omer" you'd add inline signature blocks (similar to the existing comment marker pattern) — straightforward but separate work.

6. What this tier explicitly does NOT solve

Per-recipient revocation. If Claude is compromised, you must rotate the workspace passphrase. There's no way to revoke just Claude while keeping Omer's access — both share the same MK. Tier 5 (per-recipient pubkey) is what you'd add later if you need this.
Workspace passphrase compromise = full read access. Anyone with the workspace passphrase can read every file in the workspace. Signing keys protect who can write a valid signature, not who can read.
Forward secrecy on revocation. Past file versions stay readable by anyone who already had the passphrase and kept a copy. Encryption can't unsend.

7. Upgrade path to Tier 5

The file format is designed to grow. The single wrapped_CEK field becomes wrapped_CEK[N]: one wrap per recipient public key. Workspace master key is replaced by per-recipient X25519 wrapping. Old Tier-3 files keep working because the version byte distinguishes formats. No body re-encryption needed — only the small key-wrap section grows.
